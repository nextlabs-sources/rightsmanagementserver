Manage Service Providers

This page displays three sections:

 

  • Configured Providers - Displays a list of available service providers that are integrated with Rights Management Server (RMS). This allows  users to add repositories for the given service providers in RMS.
  • Available Providers - Displays a list of available service providers that RMS supports. You can configure these service providers to integrate with RMS to grant users access to the repository from RMS. After the service providers are configured, it will move to the Configured Providers section. Currently, RMS supports Google Drive, OneDrive, SharePoint Online, SharePoint On Premise, and Dropbox service providers.
  • Cross-Launch Apps - Displays a list of service providers that can be integrated with RMS  to allow users to view the rights protected or unprotected files directly from these service providers. This feature is supported only by SharePoint On-Premise and SharePoint Online apps.
Dropbox App Settings

  1. In the Available Providers section, hover the mouse pointer over Dropbox.

  2.  Click Configure.

  3. In the Dropbox app settings, enable Allow personal repositories for users to add their personal repositories.

  4. Enter the following details.

    NOTE:  The following settings can only be obtained after a Dropbox App has been created for Rights Management Server. For more details, refer to the Rights Management Server Administration Guide.

    Field Description

    App Key

     

    This value is used by Rights Management Server to establish a secure communication channel with Dropbox.

     

    App Secret

     

    This value is used by Rights Management Server to establish a secure communication channel with Dropbox

     

    Redirect URL

     

    The URL to access the Rights Management Server portal

     

  5. Click SAVE.

SharePoint App Settings

  1. In the Available Providers, hover the mouse pointer over SharePoint.

  2. Click Configure.

  3. Enable SharePoint to allow users access to the SharePoint repository.

  4. Enable Allow personal repositories to allow users to add their own personal repository. If it is not enabled, the repository will be shared by all users.
    NOTE: If the Allow personal repositories option is enabled, only the administrators will have the option to share the repository with other users from the Manage Repositories menu.

  5. Click SAVE.

SharePoint Cross-Launch App Settings

  1. In the Cross-Launch Apps, hover the mouse pointer over SharePoint Cross-Launch.

  2. Click Configure.

  3. Enter the following details:

    Field Description

    App Name

     

    App name for the SharePoint App you want to configure.

     

     

    App Key

     

    This is the value generated by SharePoint when you registered your SharePoint Application. For more details, see Registering your SharePoint Application in the Rights Management Server Administrator Guide.

    App Secret

    This is the value generated by SharePoint when you registered your SharePoint Application. For more details, see Registering your SharePoint Online Application in the Rights Management Server Administrator Guide.

    Remote Web Application URL

     

    This is the URL of your SharePoint On-Premise web application that you have deployed on your IIS. You must append the following to it:

     

    /Pages/Default.aspx

     

    For example if your SharePoint Site App URL is:

    https://myserver:4444

    then you must type it as:

    https://myserver:4444/Pages/Default.aspx

    NOTE: If you have not changed the default port number (443)for your server then you don’t need to specify the port number in the URL. Therefore, the above URL would be listed

    as: https://myserver/Pages/Default.aspx

     

    Redirect URL

    The URL of the server hosting Rights Management application.

    App Menu Display Text

    The menu name displayed in the SharePoint app for viewing a file using Rights Management

  4. Click SAVE.
